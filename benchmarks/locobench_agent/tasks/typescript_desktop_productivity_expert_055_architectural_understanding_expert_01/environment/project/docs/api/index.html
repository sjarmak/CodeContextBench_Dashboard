    Getting Started

    PaletteFlow Studio exposes a modular TypeScript SDK that lets you write “palette plugins”—small bundles that register new node types, importers, commands, renderers, and themes. Plugins are sandboxed via Context Isolation, guaranteeing that poorly-behaved extensions cannot corrupt user workspaces.

    Every plugin must export a default function that receives a PluginContext—your gateway to the running application— and returns a clean-up callback. The context surfaces domain services, event observables, and dependency injection helpers.

    Core Entities

    The following interfaces live in the @paletteflow/core package and are framework-agnostic. 

    
/**
 * A Canvas is an infinite 2-D plane that groups nodes.
 */
export interface Canvas {
    readonly id: CanvasID;
    title: string;
    viewport: Viewport;
    nodes: ReadonlyArray;
    themeId: ThemeID;

    linkNode(child: NodeID, parent: NodeID): void;
    unlinkNode(child: NodeID, parent: NodeID): void;
    moveNode(id: NodeID, position: Vec2): void;
    clone(recursive?: boolean): Canvas;
}

    Plugin API

    Import the SDK from the global package: 

    
import { definePlugin, z } from '@paletteflow/sdk';

export default definePlugin(({ canvasService, events, storage }) => {
    // Register a new node definition
    canvasService.registerNodeType({
        kind: 'todo',
        title: 'To-Do List',
        icon: 'check-square',
        schema: z.object({
            title: z.string().default('Untitled'),
            items: z.array(z.string()).default([])
        }),
        render: async (node, ctx) => {
            const div = document.createElement('div');
            div.innerHTML = \`
                <h3>\${node.data.title}</h3>
                <ul>\${node.data.items.map(i => '<li>' + i + '</li>').join('')}</ul>
            \`;
            return div;
        }
    });

    // Listen for crash analytics
    events.crashReport.pipe().subscribe(report => {
        storage.append('crashes', report);
    });

    // Teardown callback
    return () => {
        canvasService.unregisterNodeType('todo');
    };
});

    Use-Cases

    Each use-case is a focussed application service following the Command Pattern. They are exposed through the dependency-injection container, ensuring that plugins and UI layers never reach into business logic directly. For example, CreateNode orchestrates validation, authorization, and persistence before emitting a domain event: 

    
// Pseudo-simplified example
export class CreateNode {
    constructor(
        private readonly repo: NodeRepository,
        private readonly bus: DomainEventBus
    ) {}

    async execute(request: CreateNodeRequest): Promise {
        const node = Node.create(request); // domain factory, performs invariants
        await this.repo.save(node);
        this.bus.publish(new NodeCreated(node));
        return node;
    }
}

    Event Bus

    Plugins never poll— they subscribe to strongly-typed observables powered by rxjs. The example below shows how to react to node selection: 

    
import { definePlugin } from '@paletteflow/sdk';

export default definePlugin(({ events, commandPalette }) => {
    const sub = events.nodeSelected.subscribe(({ node }) => {
        commandPalette.info(\`Selected node: \${node.title}\`);
    });

    return () => {
        sub.unsubscribe();
    };
});
